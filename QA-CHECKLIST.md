# PMP模擬試験 音声読み上げ QAチェックリスト

## テスト環境
- [ ] PC Chrome
- [ ] iPhone Chrome（スリープ状態）
- [ ] iPhone Safari（スリープ状態）

## 1. 音声再生の基本動作
- [ ] 🔊ボタンを押すと音声が再生される
- [ ] 「問題N番」と問題番号を読み上げてから問題文を読む
- [ ] 問題文の後に1.5秒の間がある
- [ ] 「正解は〇〇」と正解を読み上げる
- [ ] 正解の後に1.5秒の間がある
- [ ] 「解説です。〇〇」と解説を読み上げる

## 2. 問題間の遷移
- [ ] 1問目の解説が終わったら自動で2問目に進む
- [ ] 問題間に3秒の無音（SSMLのbreak）がある
- [ ] 最後の問題まで途中で止まらず自動で進む
- [ ] 最後の問題が終わったら再生が停止する

## 3. スリープ状態での動作（iOS重要）
- [ ] スリープ中も音声が途切れず再生される
- [ ] スリープ中も次の問題に自動で進む
- [ ] スリープ中もTTSエラーが出ない
- [ ] ロック画面に再生コントロールが表示される

## 4. 停止・操作
- [ ] 🔊ボタンを再度押すと停止する
- [ ] 再生中に問題番号ドットを押すとその問題にジャンプして再生が続く（再fetchなし）
- [ ] 再生していない時も問題番号ドットで移動できる
- [ ] ホームに戻ると音声が停止する

## 5. BGM
- [ ] BGMトグルONで再生される
- [ ] BGMの音量が音声より十分小さい（音声が聞き取れる）
- [ ] BGMトグルOFFで停止する

## 6. UI
- [ ] プログレスバーが表示され残り秒数が正しく表示される（実際の再生時間と一致）
- [ ] 秒数が0になるまでカウントダウンされる
- [ ] 音量調整UIが再生中に表示される
- [ ] 最終更新時間がメインページに表示される

## 7. エラーハンドリング
- [ ] APIキー未入力時にプロンプトが表示される
- [ ] 不正なAPIキーでわかりやすいエラーが出る
- [ ] ネットワークエラー時にリトライされる

## 既知のiOS制限事項
- iOSではAudio.volumeが効かない（端末ボリュームで調整）
- iOSスリープ中は新しいAudio要素のplay()ができない
  → 対策：全問題の音声を1つのAudioで連続再生する
